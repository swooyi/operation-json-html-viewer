문항뷰어 사용 매뉴얼

처음 사용하는 순서

문항뷰어는 왼쪽 문항 목록에서 문항을 고르고 오른쪽 상세 화면에서 내용과 이미지를 확인하는 화면입니다.

문항 선택 → 내용·이미지 확인 → JSON/이미지 수정 → 다시 읽기(R) → 오류 확인
작업 중 JSON 파일이나 이미지 파일을 수정한 경우, 다시 읽기 또는 R 키로 변경 내용을 다시 불러올 수 있습니다.
1. 빠른 시작

왼쪽 목록에서 JSON 파일과 section을 펼친 뒤 확인할 문항을 선택합니다. 폴더를 바꾸고 싶을 때는 상단의 폴더 선택 또는 최근 폴더에서 작업 폴더를 다시 선택할 수 있습니다.

왼쪽에서 문항을 선택하면 오른쪽에서 문제, choices, answer_type, answer, 해설, 이미지, JSON 원문을 확인할 수 있습니다.

전달받은 작업 폴더의 구조를 유지한 상태로 사용해 주세요. JSON 안의 [img: 파일명] 정보와 실제 이미지 파일이 연결되어야 이미지가 정상 표시됩니다.
  1. 왼쪽 목록에서 JSON 파일, section, 문항 번호를 차례로 확인합니다.
  2. 확인할 문항을 클릭합니다.
  3. 오른쪽 상세 화면에서 문제, 보기, 정답, 해설, 이미지 표시 상태를 확인합니다.
2. 화면 구성

화면은 상단 작업 영역, 왼쪽 문항 목록, 오른쪽 상세 화면으로 나뉩니다.

항목설명
홈 아이콘메인페이지로 돌아갑니다.
?문항뷰어 사용 매뉴얼을 엽니다.
폴더 선택JSON 파일과 이미지 파일이 들어있는 폴더를 처음 불러옵니다.
최근 폴더이전에 열었던 폴더 목록을 다시 엽니다. 브라우저 권한이 필요할 수 있습니다.
다시 읽기현재 폴더를 다시 불러와 JSON 파일 수정, 이미지 파일 추가, 이미지 파일 수정, 이미지 파일 교체 내용을 갱신해서 확인합니다. 작업 중 파일을 고친 뒤 화면에 바로 반영하고 싶을 때 사용합니다. 단축키는 R입니다.
상단 개수 표시파일 N개 / section N개 / 문항 N개 / 이미지 N개 형식으로 현재 불러온 데이터를 보여줍니다.
왼쪽 목록JSON 파일, section, 문항 번호 순서로 표시됩니다.
오른쪽 상세 화면선택한 문항의 문제, choices, answer_type, answer, 해설을 보여줍니다.

최근 폴더는 이전에 열었던 폴더를 다시 선택하는 기능입니다. 브라우저에 따라 폴더 기억을 지원하지 않을 수 있으며, 권한이 끊기면 최근 폴더 권한이 필요합니다. 폴더 선택을 다시 눌러 주세요.가 표시될 수 있습니다.

3. 왼쪽 문항 목록

왼쪽 목록은 JSON 파일, section, 문항 번호 순서로 표시됩니다. 기본 상태에서는 목록이 접혀 있으므로 필요한 파일과 section만 펼쳐서 확인합니다.

항목설명
문항 찾기파일명, section, 문항 번호, 문제, 보기, 정답, 해설, image_edit_notes 내용을 검색합니다.
필터이미지 있는 문항, blocks 있는 문항, 이미지 누락 문항, LaTeX 오류 문항을 골라 봅니다.
전체 펼치기왼쪽 목록의 파일과 section을 모두 펼칩니다.
전체 접기왼쪽 목록의 파일과 section을 모두 접습니다.
4. 오른쪽 상세 화면

왼쪽 목록에서 JSON 파일명을 펼치면 section 목록이 보입니다. section을 펼치면 1번, 2번 같은 문항 번호가 표시됩니다. 문항을 클릭하면 오른쪽 상세 화면에서 해당 문항을 확인합니다.

항목의미
문제JSON의 question.stem 내용입니다.
blocksJSON의 question.blocks가 있을 때 표시됩니다.
choices선택지입니다. ①, ②, ③ 형식으로 표시됩니다.
answer_type정답 유형입니다.
answer정답 값입니다.
해설문항 해설입니다.
image_edit_notes이미지 작업 메모입니다. 기본은 접힌 상태이며 펼치기로 확인합니다.
JSON 원문 보기 ON/OFF현재 문항의 원본 JSON을 그대로 확인합니다.
  • 이전문항: 현재 필터와 검색 결과 안에서 이전 문항으로 이동합니다.
  • 다음문항: 현재 필터와 검색 결과 안에서 다음 문항으로 이동합니다.
  • Alt+Q: 이전 문항으로 이동합니다.
  • Alt+W: 다음 문항으로 이동합니다.
5. 검색과 필터

문항 찾기에는 파일명, section, 문항 번호, 문제, blocks, choices, answer, 해설, image_edit_notes에 포함된 내용을 검색할 수 있습니다.

LaTeX 명령을 찾을 때는 \frac, ₩frac, \dfrac, ₩dfrac 같은 입력도 사용할 수 있습니다.

색상필터설명
초록img 있는 문항[img: 파일명] 또는 image_edit_notes가 있는 문항만 표시합니다.
보라blocks 있는 문항question.blocks가 있는 문항만 표시합니다.
노랑이미지 파일을 찾을 수 없는 문항JSON에는 이미지 참조가 있지만 실제 이미지 파일을 찾지 못한 문항만 표시합니다.
빨강latex 오류있는 문항$, $$, \(, \), \[, \] 구분자 개수가 맞지 않는 문항만 표시합니다.
6. 이미지 확인

이미지는 JSON 본문 안의 [img: 파일명] 형식을 기준으로 표시됩니다. 뷰어는 선택한 폴더 안의 png, jpg, jpeg, webp, gif 파일을 찾아 이미지 목록을 만들고, JSON에 적힌 이미지 파일명과 연결합니다.

기능설명
이미지 표시체크하면 [img: 파일명] 위치에 이미지를 보여줍니다. 해제하면 원문 텍스트로 봅니다.
이미지 크기표시되는 이미지 크기를 100px-500px까지 조절합니다.
이미지 없음: 파일명JSON에는 이미지가 적혀 있지만 실제 파일을 찾지 못한 상태입니다.
파일명 복사image_edit_notes의 image 값에서 확장자를 제외한 파일명을 복사합니다.
이미지가 보이지 않을 때는 먼저 파일명이 정확한지 확인해 주세요. 띄어쓰기, 확장자, 폴더 위치가 달라지면 이미지 없음: 파일명으로 표시될 수 있습니다.
7. 단축키
단축키동작
Alt+Q현재 필터와 검색 결과 안에서 이전 문항으로 이동합니다.
Alt+W현재 필터와 검색 결과 안에서 다음 문항으로 이동합니다.
R현재 선택된 폴더를 다시 읽습니다. 검색창이나 입력창에 포커스가 있을 때는 실행되지 않습니다.
8. 오류와 경고

상단에 오류 N개가 표시되면 JSON 문법 오류, 넘버링 오류, LaTeX 경고가 있다는 뜻입니다. 오류 N개를 클릭하면 오류만 보기 ON / 오류 N개 상태가 되며, 오류가 있는 항목만 왼쪽 목록에 표시됩니다.

표시의미확인 방법
JSON 첫 번째 감지 오류JSON 파일을 파싱하지 못했습니다.오른쪽의 오류 메시지, 줄, 칸, 주변 원문을 확인합니다.
JSON 최상위 값은 객체여야 합니다.JSON 최상단이 객체 형식이 아닙니다.파일 구조가 { ... } 형태인지 확인합니다.
sections 배열을 찾을 수 없습니다.JSON에 sections 배열이 없습니다.생성된 JSON 구조를 확인합니다.
넘버링 오류한 파일 안의 q_number가 1부터 문항 개수까지 순차적으로 이어지지 않습니다.section이 바뀌어도 번호가 이어지는지 확인합니다.
이미지 파일을 찾을 수 없음: 파일명이미지 참조는 있지만 파일 연결에 실패했습니다.실제 이미지 파일 위치와 파일명을 확인합니다.
$ 구분자 개수가 맞지 않습니다.LaTeX 인라인 수식 구분자가 맞지 않습니다.해당 문항의 수식 기호를 확인합니다.
\( \) 구분자 개수가 맞지 않습니다.LaTeX 괄호형 수식 구분자가 맞지 않습니다.여는 기호와 닫는 기호를 확인합니다.
\[ \] 구분자 개수가 맞지 않습니다.LaTeX 블록 수식 구분자가 맞지 않습니다.여는 기호와 닫는 기호를 확인합니다.
JSON 파서는 파일당 첫 번째로 감지한 문법 오류만 확정 표시합니다. 이 오류를 수정한 뒤 다시 읽기를 누르면 다음 오류가 표시될 수 있습니다.
